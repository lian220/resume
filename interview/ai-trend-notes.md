## Part 1. 트렌드 한눈에 (2025 → 2026)

- **금지 → 허용**으로 전환 중. "빠른 구현"은 AI가 하니까, 회사는 **AI 출력을 검증·설명·통제하는 능력**을 본다.
- 평가 키워드 = **AI fluency = 프롬프트 엔지니어링 + 출력 검증 + 디버깅**.
- **시스템 설계가 최대 변별점**. 구현은 베이스라인, 시니어는 **아키텍처·트레이드오프·"AI를 지휘"**하는지.
- 배경: LLM이 취약/버그 코드를 **9.8~42.1%** 생성 → "검증할 줄 아는 사람"이 가치. NY 기준 ~25% 기업이 AI 허용(→50% 전망).

## Part 2. 4가지 형태별 — 무엇을 보고, 어떻게 준비

### ① 라이브 AI 페어코딩 (Meta형)
- **루브릭 4축**: 문제해결 / 코드품질 / **검증**(테스트·엣지케이스·AI출력 리뷰) / **커뮤니케이션**(내레이션)
- **탈락 사유**: 코드베이스 안 읽고 시작 / "프롬프트로 때우기"(이해 없이) / 무검증 복붙 / 침묵 / 시간배분 실패
- **프롬프팅**: 한 번에 맞히게 **풀 컨텍스트 + 구체 요구**. ❌"이 문제 풀어줘" → ✅"insert·prefix search 지원하는 trie 클래스 구현해줘". AI는 보일러플레이트·자료구조·복잡도·헬퍼에만, **핵심 알고리즘은 본인**.
- **핵심 스킬**: "낯선 코드 빨리 읽기"(Meta가 가장 중요하다고 명시) + **이중 대화**(AI+면접관 동시 내레이션) 연습.

### ② 기획서(PRD/스펙) → 구현 (spec-driven)
- **무엇을**: 모호한 요구를 **명세로 구조화 → 설계 → 구현**, 명세를 **실행 가능한 검증 게이트**로 쓰는지.
- **준비**: 요구사항에서 **엣지케이스·비기능요구(동시성·멱등·실패처리)를 먼저 끌어내** 질문하는 습관.

### ③ AI 허용 과제전형/홈테스트
- **무엇을**: AI를 **어디에 쓰고 어디에 안 썼는지**의 판단. 전부 AI ❌, **잘 정의된 하위작업에만 전략적으로**, 전체 통제는 본인.
- **준비**: 결과물에 **"왜 이렇게/AI를 어디에 썼나" 의사결정 노트** 동봉.

### ④ "AI 어떻게 쓰냐" 행동·심층
- **무엇을**: 실제 활용 사례 + **한계 인식**(어디서 AI가 틀렸고 어떻게 잡았나).
- **준비**: 구체 일화 1~2개(아래 Part 3 스크립트).

### 시니어 백엔드 차별화
- 구현 속도 자랑 ❌ → **설계·트레이드오프·검증을 AI와 어떻게 분업**했는지 ✅
- AI를 **주니어 페어처럼 지휘**: 명세 주고 → 결과 비판적 리뷰 → 엣지케이스로 추궁 → 최종 책임은 본인
- "최적화 포인트는 알고리즘이 아니라 **테스트 데이터 특성**에 숨어있다"(Meta) — 데이터/숨은 함정 읽는 눈

## Part 3. 행동면접 답변 스크립트 (본인 STAR 4 추천인 시스템 기반)

> 말하기용. 두괄식 → 사례 → 한계/배운점. 1~1.5분 분량.

### 스크립트 A — "AI를 실무에서 어떻게 쓰고, 한계는 어떻게 다루나?"
> **(두괄식)** "저는 AI를 **주니어 페어처럼** 씁니다. 명세를 주고, 결과는 **가설로 받아 코드·도메인 정책으로 반증**하고, 최종 판단은 제가 합니다. 특히 'AI가 자신 있게 틀리는 지점을 잡아내는 것'을 제 역할로 봅니다."
>
> **(사례)** "최근 멤버십 **추천인 리워드 시스템**을 점검하면서 AI로 코드 분석과 보안 리뷰를 돌렸는데, 두 가지를 직접 바로잡았습니다.
> 첫째, 문서·AI 요약은 할인 하한을 '1원 보정'이라 했는데 **실제 코드는 0원 클램프**였어요 — PG가 1원 결제를 거부하거든요. 문서와 구현의 드리프트를 코드로 확인해 정정했습니다.
> 둘째, AI 보안 리뷰가 '추천인당 캡이 없다 = 무한 파밍, High'라고 띄웠는데, 저는 **오프레이밍이라 판단**했습니다. 한 추천인을 여러 명이 지정해 리워드를 여러 번 받는 건 버그가 아니라 **어필리에이트 그로스의 의도된 동작**이니까요. 진짜 리스크는 '가짜 피추천인 자가파밍'으로 좁혀서, 대응도 블랭킷 캡이 아니라 **fraud 군집 탐지 + 클로백**으로 정리했습니다."
>
> **(한계 인식)** "AI는 **제품 의도(컨텍스트)를 모르면 그럴듯하지만 틀린 결론**을 자신 있게 냅니다. 그래서 출력은 항상 가설로 받고, 멱등 지급 같은 **핵심 설계는 AI에 안 맡기고 직접** 합니다 — 거긴 조건부 UPDATE 선점으로 정확히 1회를 제가 보장했어요."

**면접관이 파고들면**:
- *"AI를 어디까지 믿나?"* → "보일러플레이트·테스트 스캐폴딩·리팩터 후보엔 적극, **돈·동시성·보안 경계는 직접 설계+검증**."
- *"그 보안 리뷰가 맞았으면?"* → "그래서 캡 대신 진짜 리스크(가짜 피추천인+환불타이밍)는 인정하고 클로백을 백로그로 명시했습니다. AI를 무시한 게 아니라 **프레이밍을 교정**한 거죠."

### 스크립트 B — "기획서/모호한 요구를 받아 구현한 경험은?"
> **(두괄식)** "기획서를 받으면 가장 먼저 **비기능 요구와 엣지케이스를 명세로 끌어내 범위를 확정**합니다. 추천 리워드 지급 정책이 그 예입니다."
>
> **(사례)** "기획의 'D+8 지급 / 무환수'를 그대로 받지 않고 경계를 따졌습니다.
> · **지급 시점** = 첫 결제 D+8(전액환불 가능기간 종료) + 1h 안전마진 → 환불 확정 전 지급 방지.
> · **환불 처리** = D+8 이내 환불은 배치 picking 단계에서 제외(`PAYMENT_CANCEL` 검사). 지급 후 환수(클로백)는 '**현 단일 채널 스코프엔 미구현, 다채널 확장 시 30일 클로백**'으로 **범위를 명시적으로 결정**했습니다.
> · **멱등성**은 기획엔 없지만, 배치 재실행·동시성 대비 **조건부 UPDATE(`PENDING→GRANTED`) 선점**으로 정확히 1회를 보장했습니다."
>
> **(포인트)** "스펙을 **실행 가능한 검증 게이트**로 바꾼 겁니다 — 정책을 상태머신(`PENDING/GRANTED/FAILED`)과 테스트로 코드화해서, **모호함을 코드가 강제**하게."

**면접관이 파고들면**:
- *"왜 클로백을 안 했나?"* → "단일 USER_REFERRAL 채널은 D+8(전액환불기간 종료)로 충분히 방어돼서 **현 스코프 결정**이고, 다채널 확장 시점에 30일 클로백을 켜는 백로그로 문서화했습니다." (버그 아님 = 범위 결정)

### 스크립트 C — 라이브 AI 페어코딩 "내레이션 골격" (말하기 순서)
1. **(받자마자, 30초)** "먼저 코드베이스/요구를 훑겠습니다" → 진입점·자료구조·제약을 **소리내어** 파악.
2. **(구현 전)** "접근은 X, 시간복잡도 O(...), 엣지케이스 A·B·C입니다." → **설계를 먼저 선언**.
3. **(AI 호출)** 구체 요청만: "이 시그니처로 trie insert/prefix search 구현해줘" — 보일러플레이트·자료구조만.
4. **(출력 받으면)** "검증할게요" → 엣지케이스·복잡도 확인 후 채택/수정. **무검증 복붙 절대 X**.
5. **(막히면)** "프롬프트로 때우기" 대신 **핵심 로직은 직접** 작성.
6. **(시간 부족)** 못 끝내도 "남은 건 ~로 마무리" 설계를 **말로** → 이해도 점수 획득.

## Part 4. 실행 체크리스트
- [ ] CoderPad/Cursor로 **낯선 오픈소스 디버깅** 주 2~3회 (진입점→핵심클래스→데이터흐름)
- [ ] 알고리즘은 여전히 **혼자** 풀 수 있게 (backtracking·BFS/DFS·DP·trie·bitmask)
- [ ] **이중 대화** 연습: 화면공유 켜고 AI+면접관 동시 내레이션 (실제로 어렵다 — 반복 필요)
- [ ] 프롬프트 = **풀 컨텍스트 + 구체 요구 + "엣지케이스/테스트 부족분 찾아줘"** 검증 요청
- [ ] **스크립트 A·B 암기** (위 STAR 4 사례) — 숫자(D+8·0원·999개월) 정확히
- [ ] 타겟 회사 **AI 허용 여부 지원 전 확인** (국내 대형은 아직 금지 다수)
- [ ] 과제전형이면 **의사결정 노트** 동봉(어디에 AI 썼고 왜)
