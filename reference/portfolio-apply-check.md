# 영상 전략 → 내 실제 프로젝트(STAR) 적용 체크

> 기준: `포트폴리오-주제선정-전략-요약.md` (코딩하는기술사 영상)
> 작성: 2026-06-07 · 용도: **앞으로 할 일** — 실무 프로젝트를 영상 관점으로 포트폴리오화
> 전제: 선생님은 실무 시니어 → 영상의 유형 ⑤(실사용자 운영)는 **이미 최상위로 충족**. 갭은 "어필 방식"에 있음.

## 1. STAR ↔ 영상 유형 매핑 + 강점/보강

| STAR | 해당 영상 유형 | 이미 충족 (강점) | 보강하면 강해지는 것 |
|---|---|---|---|
| **1. 인증·인가 (게이트웨이 RBAC)** | ③ 빅테크 패턴 재현급 + ⑤ 실운영 | 트레이드오프 多(자체 JWT·Redis vs Caffeine·게이트웨이 중앙집행), 공학적 근거(OWASP API1/3, TTL 1분/10분), 전환율 30.2% | **아키텍처 다이어그램**(게이트웨이+6서비스), **② AI 활용 서사** 부재, 인증 엔드포인트 **부하테스트 수치(P95/TPS)** |
| **2. SigNoz 관측성** | ① 일상 문제(디버깅 병목)에서 출발 + 도메인 관찰력 **최상** | 영상 4대 방향성 1·2번 그 자체(문제발견→제안→도입→표준화), 벤더락인·비용 트레이드오프 | **SLO/에러버짓**(영상이 강조한 공학적 기준!), 비용절감·디버깅시간 **수치 빈칸**, 설명 영상 |
| **3. 소통관리 도메인** | ⑤ 실사용자(본부↔점주) + 도메인 관찰력 | 실운영, N+1 튜닝(공학적), 양방향 도메인 설계 | **설계 결정 명시**(STI 등), P95 before/after **수치 빈칸**, 트레이드오프 기록 |
| **4. 멤버십 결제+추천인** | ⑤ 매출 직결 실서비스 + ① 결제는 누구나 이해 | 멱등성·상태머신·어뷰징 방어(공학적 깊이 최상), 실코드 기반, Strategy 회피 근거 | 결제 성공률·일배치 건수 **수치 빈칸**, ② AI 활용 서사, 부하테스트 |

> **요약**: 4개 STAR 모두 영상이 말하는 "단순 CRUD"를 한참 넘었음(설계·트레이드오프·수치 다 있음). **부족한 건 실력이 아니라 ① 수치 빈칸 ② 시각 자료(다이어그램/영상) ③ AI 활용 서사** 세 가지뿐.

## 3. 공통 보강 액션 (우선순위)

1. **🥇 아키텍처 다이어그램 4장** — 영상이 꼽은 최대 갭. 사내 코드 비공개여도 다이어그램은 공유 가능.
   - S1: 게이트웨이 중앙 RBAC + 6서비스 + Redis (이미 `tech-review.html` 클라우드 검토에 ASCII 토폴로지 있음 → 그림화)
   - S2: OTel Agent → Collector → ClickHouse → 대시보드/알람
   - S4: 빌링키 발급 → 일배치 → 상태머신 → 리워드 D+8
2. **🥈 수치 빈칸 채우기** — `PROGRESS.md` 남은 빈칸(S2/S3/S4). 영상 권고대로 **k6 부하테스트로 P95/TPS** 직접 생성 가능(사내 유사 재현). 이미 `k6-prom-grafana-script` 보유.
3. **🥉 프로젝트 설명 영상 1편** — 영상이 "포트폴리오보다 강력"하다고 강조. 후보: S1(게이트웨이 RBAC) 또는 S4(결제 멱등). "문제→설계 의도→아키텍처→결과" 구조.
4. **AI 코딩 활용·검증 서사 (유형 ②)** — 현재 STAR엔 없음. "AI로 구현하되 할루시네이션·회귀버그·보안을 직접 검토·개선한 경험"을 별도 한 꼭지로. 시니어 차별화 포인트.
5. **SLO/에러버짓 (S2)** — 영상이 강조한 "공학적 기준". 2차 검토 Critical과도 일치 → 1석2조.
6. **블로그/유튜브 지속 기록** — 단발 포트폴리오보다 강력. 단 얕은 글·방치는 역효과(영상 경고).

## 4. "앞으로 할 일" 백로그 (영상 2편 + 2차 검토 통합)

### A. 글쓰기 — PAR 구조 강화 (영상 2 `이력서-PAR구조-작성법-요약.md` 기반)
- [ ] **자기소개/헤드라인을 숫자 후킹형으로** (예: "정회원 전환율 30.2%·결제 멱등 시스템을 설계한 백엔드…") → 이력서 전체 컨셉 일관성
- [ ] **각 STAR에 "왜 그 설계를 택했나"(Analyze/트레이드오프) 1줄** — ※ 2차 검토 논리균열 #5("강점만 나열")와 **동일 과제**
- [ ] **action만 나열된 항목에 Problem(내 고민)을 드러내기** — "왜 했는지"를 이력서가 말하게
- [ ] **수치 문장에 측정 과정·결과까지** — 단순 "N% 개선"❌ → "어떻게 측정했고 어떤 수치가 나왔는지"

### B. 자료 — 시각화·검증 (영상 1 주제선정 기반)
- [ ] 아키텍처 다이어그램 4장 (S1/S2/S3/S4)
- [ ] k6 부하테스트로 S1 인증·S4 결제 P95/TPS 수치 확보
- [ ] 프로젝트 설명 영상 1편 (S1 or S4)

### C. 수치·설계 보강
- [ ] PROGRESS 남은 수치 빈칸 채우기 (S2/S3/S4)
- [ ] AI 코딩 활용·검증 경험 → 별도 STAR/꼭지로 작성 (유형 ② / 면접 단골 질문 대비)
- [ ] S2 SLO/SLI·에러버짓 정의 (2차 검토 Critical과 통합)
- [ ] S3 설계 결정(STI·thread aggregate·Outbox) 명시 — `[확인]` 코드 검증 후 확정

### D. 장기
- [ ] 기술 블로그/유튜브 정기 기록 루틴

> 핵심: 두 영상 + 2차 전문가 검토가 모두 같은 결론 → **"각 STAR에 왜 그 설계를 택했나(A) 1줄 + 수치(R)"**. 선생님은 콘텐츠(실력·실운영)는 충분하니 **포장(PAR 서술·다이어그램·수치·영상·AI서사)** 만 채우면 됨.
