1

사용자 인증·인가 시스템 풀스택 (SNS 로그인 + RBAC)

P문제
차별화상회는 ID/PW 6단계 가입만 있어 모바일 가입 마찰이 최대 이탈 병목. ROS 백오피스는 권한이 코드 곳곳에 흩어져 본부·점주·관리자 통제 불가.
A분석
외부 IdP 종속 대신 토큰을 자체 발급·통제하기로 결정. 권한은 stale·위조를 감수하더라도 게이트웨이 중앙 집행 + 서버사이드 이중검증이 맞다고 판단.
A실행
OAuth 2.1 PKCE(S256) → 자체 JWT + Refresh Token Rotation. RBAC 3단 모델(ROOT/FUNCTION/ENDPOINT) + 객체레벨 인가, 게이트웨이 중앙 RBAC.
R결과
신규 가입 일 10 → 150명(15배), 정회원 전환율 30.2%, 인증 운영장애 0건, 다운스트림 6서비스 JWT 재파싱 제거.
2

멤버십 자동결제(토스 빌링) + 추천인 그로스 루프

P문제
카드 1회 등록 → 매월 자동결제 + 가입·연체·해지 라이프사이클이 필요. 돈이 걸린 흐름이라 중복 결제·이중 지급 = 즉시 사고.
A분석
카드 정보 자체 보관(PCI 부담)을 회피하기로 결정. 중복 방지는 분산락 대신 DB 멱등성, 리워드는 자동 재시도의 이중 지급 위험을 인지.
A실행
토스 빌링키만 저장(vault 위임). idempotency 키 + 조건부 UPDATE로 중복 차단, 리워드는 실패 격리 + Slack 알림 + 수동 멱등 재처리.
R결과
멤버십 자동결제 BE 단독 출시, 결제·환불·리워드 회수 흐름 회귀 0건. [확인 필요: 결제 성공률 N%, 추천 경유 가입 N%]
3

본부↔점주 양방향 소통관리 도메인 신규 구축 (외식UP ROS)

P문제
ROS 1.0 출시에 본부-점주 채널이 부재 → 공지 미전달·동의서 누락 리스크. 공지/동의서/공문 3종 + 알림톡 + 1:1 문의가 묶인 도메인 필요.
A분석
타입별 테이블 분리 대신 공통 라이프사이클 공유 모델로 결정. 알림톡은 단순 retry 시 dual-write 유실 위험이 있다고 판단.
A실행
STI(단일 테이블 상속) 모델링으로 폴리모픽 목록 최적화, 알림톡에 Transactional Outbox 적용. 출시 후 N+1 제거 + 복합 인덱스.
R결과
제로에서 39건 API BE 단독 구축(본부 26 + 점주 13, 전건 배포), QA 이슈 14건 전건 종결. [확인 필요: 구축 기간, 목록 P95 개선치]
4

관측성(Observability) 자발 도입 → 팀 표준 정착

P문제
운영 장애를 로그 grep으로만 추적 → 디버깅 병목. 팀 관측성 표준 자체가 부재, SaaS(Datadog/New Relic)는 비용 + 데이터 외부 반출 우려.
A분석
SaaS vs OSS를 비용·데이터 주권 기준으로 비교 후 SigNoz로 결정. "관측성 다운 ≠ 서비스 다운"이라 HA는 과투자로 판단.
A실행
SigNoz self-hosted(EC2 단독 + docker-compose) 도입, OTel로 핵심 흐름만 커스텀 span(폭증 방지), traceId MDC 주입.
R결과
로그 grep → 분산 트레이스 단일 조회로 근본원인 추적, 데이터 주권 확보, 팀 표준 정착. [확인 필요: 연 절감액, 디버깅 시간 단축]
