## 주요 경험

### 1. 본부↔점주 양방향 소통관리 도메인 신규 구축 (외식UP ROS)
> ROS 1.0 신규 출시에 맞춰 **본부↔점주 양방향 소통관리**(3종 컨텐츠 + 1:1 문의 + 알림톡) 도메인을 **제로에서 BE 단독 풀스택 구축** → 출시 후 성능 튜닝까지 단독 완수

**임팩트**
- **39건 API BE 단독 구축** (본부향 26 + 점주향 13, 전건 배포완료) — 공지·동의서·공문 미전달 리스크 해소, 운영 투명성 확보
- QA 이슈 **14건 전건 종결** (첨부/업로드·권한/필터·알림톡·점주-본부 연동 안정화)
- 운영 규모/성능: *[확인 필요: 구축 기간 N개월, 목록 P95 N→Mms, 본부·점주 N명, 알림톡 월 N건]*

**핵심 의사결정**
- 3종 컨텐츠(공지/동의서/공문)를 **STI(단일 테이블 상속)**로 모델링 — 공통 라이프사이클 공유·폴리모픽 목록 조회 최적화
- 알림톡 외부 호출과 DB 트랜잭션 경계 분리 — **Transactional Outbox**로 부분 실패 시에도 게시글 상태 일관성 유지(dual-write 유실 방지)
- 출시 후 실행계획 분석 → **N+1 제거 + 복합 인덱스 + Fetch Join**으로 목록/상세 성능 개선

### 2. 멤버십 자동결제(빌링) 시스템 + 추천인 그로스 루프 (차별화멤버십)
> **토스페이먼츠 빌링키 기반 정기결제 시스템**을 BE 단독 설계·구현 — 라이프사이클 상태머신, 실패 재시도, 안분 환불, 무료체험, ERP 회계 연동, 추천인 그로스 루프까지 **결제 도메인 풀스택**

**임팩트**
- **차별화멤버십 자동결제 BE 단독 출시** — 카드 1회 등록으로 매월 자동결제, 운영 개입 없이 안정 운영
- 결제·환불·리워드 회수 흐름 **회귀 0건** — 결제 도메인 안정성을 매출 안정성으로 직결
- *[확인 필요: 카드 등록 회원 결제 성공률 N%, 추천코드 경유 신규 가입 N%]*

**핵심 의사결정**
- **PCI DSS 인지** — 카드 정보는 시스템에 일절 미저장, 토스 빌링키만 보관(vault 위임)
- 일배치에 **idempotency 키**(`memberId+billingDate` 유니크)로 재실행 시 중복 결제 차단, API/DB 트랜잭션 경계 분리로 외부 지연에도 일관성 유지
- 추천 리워드 **D+8 멱등 지급**(조건부 `UPDATE PENDING→GRANTED`로 정확히 1회) + 환불 시 지급 제외 — **돈이 걸린 흐름은 blind auto-retry 대신 실패 격리 + 수동 게이트**

### 3. 사용자 인증·인가 시스템 풀스택 설계 (SNS 로그인 + RBAC)
> 모바일 소셜 로그인(Kakao·Apple OAuth + PKCE)부터 B2B 백오피스 RBAC까지 — 두 프로덕트의 **인증·인가를 동일 원칙으로 설계·배포**

**임팩트**
- **신규 가입자 일평균 10명 → 150명 (15배 / +1,400%)** — 6단계 가입 흐름을 3단계로 단축, 모바일 가입 마찰 제거
- 준회원 → 정회원 **전환율 30.2%** (사전등록 9,723건 중 2,936건) · 인증 관련 운영 장애 **0건**

**핵심 의사결정**
- OAuth 2.1 **PKCE(S256) → 자체 JWT + Refresh Token Rotation** — 구 토큰 재사용 시 세션 전체 무효화로 replay 방어
- **RBAC 3단 모델(ROOT/FUNCTION/ENDPOINT) + 객체레벨 인가** — 수직 권한 상승 차단(OWASP API1/API3, IDOR 방어)
- **API 게이트웨이 중앙 RBAC 집행** — 다운스트림 6개 서비스의 JWT 재파싱 제거, Redis 분산 캐시 + 즉시 무효화 API

### 4. 관측성(Observability) 시스템 자발 도입 → 팀 표준 정착
> 운영 디버깅 병목을 스스로 발견 → OpenTelemetry 기반 **self-hosted SigNoz 제안·도입** → **팀/전사 관측성 표준 정착**

**임팩트**
- **로그 grep 의존 → 분산 트레이스 단일 조회**로 외부 호출 지연·N+1·배치 병목 근본원인 즉시 추적
- **데이터 주권 확보** — 사용자/매출 trace가 모두 사내 인프라에 머무름, 벤더 종속 회피
- *[확인 필요: SaaS 대비 연 N만원 절감, 디버깅 N시간→M분, 확산 N명/N개 서비스]*

**핵심 의사결정**
- SaaS(Datadog·New Relic) vs OSS **비용·데이터 주권 비교 후 SigNoz 채택 설득** — 의사결정을 직접 주도
- **관측성 다운 ≠ 서비스 다운** 판단으로 단일 EC2 + docker-compose 선택(HA 비용 절감)
- 핵심 비즈니스 흐름에만 커스텀 span 선별 적용(span 폭증 방지), traceId MDC 주입으로 로그↔Trace 연관
